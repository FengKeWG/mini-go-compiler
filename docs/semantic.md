# MiniGo 语义分析设计

语义分析阶段采用“递归下降分析 + 语义动作”的方式实现。语法分析器识别语法成分时，语义分析器执行对应动作：填写符号表、分配活动记录地址、检查简单语义错误，并生成四元式。

## 符号表设计

程序内部用一个总表保存所有符号，对应 PPT 中的 `SYNBL`：

```txt
NAME      名字
TYPE      类型，变量/常量记录自身类型，函数记录返回类型
CAT       种类，f=函数，v=变量，c=常量，t=临时变量，type=类型名
ADDR      活动记录地址
```

为了让输出格式和 PPT 更一致，程序还会从总表中派生打印以下辅助表：

```txt
TYPEL   类型表，记录基本类型、数组类型、结构体类型的类型码和长度
AINFL   数组表，记录数组下界、上界、成分类型和成分长度
RINFL   结构表，记录结构体字段名、字段区距和字段类型
PFINFL  函数表，记录函数层次、入口和返回类型
CONSL   常量表，记录常量名、类型和值
LENL    长度表，记录各类型占用的值单元个数
VALL    活动记录表，记录运行时值单元的分配情况
```

说明：内部实现没有真的把每个表都写成互相指向的复杂指针结构，而是用简单的 Go 切片和映射表保存信息，打印时再整理成 PPT 的表格形式。这样代码更容易读，展示时也能对应课件。

实现中 `Symbol` 结构体额外保留 `Length` 和 `Value` 字段，方便计算地址和保存常量值；但打印 `SYNBL` 时只输出 PPT 中的四个字段。

类型长度约定：

```txt
int      4
float    8
bool     1
char     1
string   16
数组      数组长度 * 元素类型长度
结构体    所有字段长度之和
func     0
void     0
```

说明：`void` 表示函数没有返回值；函数表项的 `CAT=f`，`TYPE` 字段记录该函数的返回类型。

## 符号表底层设计

程序中使用顺序表保存 `SYNBL`：

```go
[]Symbol
```

同时使用映射表加速名字查找：

```go
map[string]int
```

其中 `map` 保存名字到符号表编号的映射。插入符号时先查映射表，若已经存在则报告重复声明错误。

数组表和结构表的底层信息也来自符号表：

```txt
数组变量：TYPE 形如 [10]int，可推出 AINFL 中的 LOW=0、UP=9、CTP=int、CLEN=4
结构类型：VALUE 形如 age:int,score:float，可推出 RINFL 中每个字段的 OFF 和 TP
```

## 活动记录设计

当前 MiniGo 只支持一个 `func main()` 层次，因此层次号 `l=0`。为了和课程 PPT 中的活动记录格式保持一致，活动记录仍然保留连接数据区和 Display 表。

```txt
0  Old SP
1  返回地址
2  全局Display地址
3  参数个数
4  Display[0]
5  局部变量区起始地址
```

变量和临时变量按照声明或生成顺序依次分配地址。常量和函数名不分配活动记录地址。

例如：

```txt
a    int   addr=5
b    int   addr=9
ok   bool  addr=13
t1   int   addr=...
```

## 四元式设计

四元式格式：

```txt
(op, arg1, arg2, result)
```

常见四元式：

```txt
(program, main, _, _)
(=, 2, _, a)
(+, a, b, t1)
(=, t1, _, a)
(<, a, b, t2)
(jfalse, t2, _, L1)
(j, _, _, L2)
(label, _, _, L1)
(return, _, _, _)
(end, main, _, _)
```

函数定义：

```txt
<函数定义> -> func <标识符> ( <参数列表> ) <返回类型> <复合语句>

语义动作：
1. 读取函数名 name。
2. 读取形参列表 params 和返回类型 returnType；若没有返回类型，则记为 void。
3. 将函数名插入符号表，种类为 f，类型为 returnType，参数信息记录在 PFINFL。
4. 将形参插入当前函数的符号表，种类为 p，并分配活动记录地址。
5. 在分析 return 语句时检查返回表达式类型是否与 returnType 兼容。
```

函数调用：

```txt
<函数调用> -> <标识符> ( <实参列表> )

语义动作：
1. 查找函数名是否已经声明。
2. 检查实参数量是否等于形参数量。
3. 逐个检查实参类型是否能赋给对应形参类型。
4. 为每个实参生成 (param, arg, 序号, _)。
5. 生成 (call, 函数名, 实参数量, result)，若函数有返回值，result 为新的临时变量。
```

## 翻译文法示例

变量声明：

```txt
<变量声明语句> -> var <标识符> <类型> ;

语义动作：
1. 读取标识符名字 name。
2. 读取类型 type。
3. 将 (name, type, v, addr, len) 插入符号表。
4. addr 按类型长度递增。
```

赋值语句：

```txt
<赋值语句> -> <左值> <赋值运算符> <表达式> ;

语义动作：
1. 检查左值是否已经声明。
2. 分析右侧表达式，得到结果位置 place。
3. 检查赋值类型是否兼容。
4. 生成四元式 (=, place, _, left)。
```

for 三段式循环：

```txt
<for循环语句> -> for <简单语句可选> ; <表达式可选> ; <简单语句可选> <复合语句>

语义动作：
1. 先分析初始化语句，生成初始化四元式。
2. 生成循环条件标签 begin。
3. 分析条件表达式，若条件为 false，生成跳转到 end 的四元式。
4. 暂存更新语句产生的四元式，等循环体结束后再放回。
5. 分析循环体，break 跳到 end，continue 跳到 post。
6. 生成 post 标签，追加更新语句四元式。
7. 更新语句执行后跳回 begin。
8. 生成循环结束标签 end。
```

自增自减语句：

```txt
<自增自减语句> -> <左值> (++ | --) ;

语义动作：
1. 检查左值是否已经声明。
2. 检查左值类型是否为 int 或 float。
3. 将 i++ 翻译成 t = i + 1 和 i = t。
4. 将 i-- 翻译成 t = i - 1 和 i = t。
```

数组声明：

```txt
<变量声明语句> -> var <标识符> <数组类型> ;
<数组类型> -> [ <整数常量> ] <类型>

语义动作：
1. 读取数组长度 len 和元素类型 elemType。
2. 数组总长度为 len * elemType 的长度。
3. 将数组变量插入符号表，并按总长度分配活动记录地址。
```

结构体声明：

```txt
<类型声明语句> -> type <标识符> struct { <字段声明表> }

语义动作：
1. 读取结构体类型名。
2. 依次读取字段名和字段类型。
3. 结构体总长度为所有字段长度之和。
4. 将结构体类型名插入符号表，种类为 type，不分配活动记录地址。
```

表达式：

```txt
<加法表达式> -> <乘法表达式> + <乘法表达式>

语义动作：
1. 分别得到左右操作数 left、right。
2. 生成临时变量 t。
3. 生成四元式 (+, left, right, t)。
4. 将 t 作为表达式结果返回。
```

条件语句：

```txt
<条件语句> -> if <表达式> <复合语句> <否则部分>

语义动作：
1. 表达式生成条件变量 cond。
2. 生成假出口标签 L1。
3. 生成 (jfalse, cond, _, L1)。
4. then 分支结束后生成跳转和 label。
```
