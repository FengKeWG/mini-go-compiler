# MiniGo 优化设计

优化阶段以语义分析生成的四元式为输入，不重新分析源程序。这样实现简单，和课程设计中“中间代码优化”的位置也一致。

## 基本块划分

基本块入口规则：

```txt
1. 第一条四元式是入口
2. label 四元式是入口
3. j、jfalse、return、end 后面的下一条四元式是入口
```

程序会输出每个基本块对应的四元式编号，例如：

```txt
B1: 四元式 1 - 31
B2: 四元式 32 - 34
```

## 常量表达式节省

如果一个四元式的两个操作数都是常量，就在编译阶段直接计算。

优化前：

```txt
(+, 2, 3, t1)
```

优化后：

```txt
(=, 5, _, t1)
```

当前支持整数、浮点数、布尔值和常见算术/关系/逻辑/位运算。

## 公共子表达式节省

在同一个基本块内，如果某个表达式已经计算过，并且中间没有修改相关变量，就复用前一次结果。

优化前：

```txt
(+, a, 1, t2)
(=, t2, _, b)
(+, a, 1, t3)
(=, t3, _, pair)
```

优化后：

```txt
(+, a, 1, t2)
(=, t2, _, b)
(=, t2, _, t3)
(=, t3, _, pair)
```

## 删除无用赋值

如果同一个基本块内某个变量被赋值后没有被使用，又被新的赋值覆盖，就删除前一次赋值。临时变量如果没有后续使用，也会删除。

优化前：

```txt
(=, 2, _, a)
(=, 5, _, t1)
(=, t1, _, a)
```

优化后：

```txt
(=, 5, _, t1)
(=, t1, _, a)
```

为了避免误删数组和结构体字段写入，当前不会删除 `scores[0]`、`stu.age` 这类赋值。

## 循环优化

当前实现的是基础“循环不变式外提”。程序识别形如下面的循环四元式：

```txt
(label, _, _, L1)
(>, a, 0, t1)
(jfalse, t1, _, L2)
...
(j, _, _, L1)
(label, _, _, L2)
```

如果循环体中有表达式每次循环计算结果都一样，就把这条表达式移动到循环入口之前。

优化前：

```txt
(label, _, _, L1)
(>, a, 0, t1)
(jfalse, t1, _, L2)
(+, limit, 1, t2)
(=, t2, _, pair)
(-, a, 1, t3)
(=, t3, _, a)
(j, _, _, L1)
```

优化后：

```txt
(+, limit, 1, t2)
(label, _, _, L1)
(>, a, 0, t1)
(jfalse, t1, _, L2)
(=, t2, _, pair)
(-, a, 1, t3)
(=, t3, _, a)
(j, _, _, L1)
```

这里 `limit + 1` 不依赖循环中被修改的变量 `a`，所以可以提前计算。为了避免误优化，当前不会外提数组元素、结构体字段、除法和取模表达式。
